# Planning Workflow (JS/TS)

## Purpose
Produce a concise, actionable implementation plan before coding.

## Plan Contents
- JIRA ID and short summary.
- Acceptance criteria (user-facing).
- TDD plan: list tests to write (unit, integration, E2E).
- API contract and data model (if applicable).
- Rollout strategy and feature flag notes (if needed).
- Estimation and risks (placeholder to fill by dev).

## Output
- `docs/plans/{JIRA}/plan.v1.md` stored in repo.
